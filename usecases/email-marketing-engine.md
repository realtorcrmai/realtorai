# Usecase: Email Marketing Engine

## Problem Statement

Realtors need to stay top-of-mind with buyers and sellers throughout long transaction cycles — sometimes years. Manual email campaigns are inconsistent, generic, and time-consuming. Without automation, a realtor misses the moment a buyer clicks a listing link at 11pm, or forgets to send a market update to their farming area, or sends the same generic newsletter to a hot buyer who is already under contract.

ListingFlow's Email Marketing Engine solves this with six event-triggered email types, Claude AI content generation, React Email HTML templates, lifecycle-driven contact journeys, granular click tracking via Resend webhooks, engagement scoring, and a visual automation builder (React Flow). Every email is personalized using the contact's `newsletter_intelligence` JSONB profile — AI reads their full click history, inferred interests, price range, and areas before generating new content.

---

## User Roles

| Role | Description |
|------|-------------|
| Realtor | Reviews AI-generated drafts in the approval queue, approves or skips them, builds automation workflows, manages segments, monitors analytics |
| Contact (Buyer/Seller) | Receives personalized emails; click and open data feeds back into their CRM profile |
| System (Cron + Webhooks) | Journey phase advancement, scheduled sends, Resend webhook processing for opens/clicks/bounces |

## Features

### 1. Six Email Types

| Type | Trigger | Content Generated by Claude |
|------|---------|---------------------------|
| `new_listing_alert` | Listing matches buyer search criteria | Area intro, listing cards (address, price, beds, baths, photo), CTA to book showing |
| `market_update` | Monthly cron schedule | Area stats (avg price, DOM, list-to-sale ratio), recent sales table, market commentary |
| `just_sold` | `listings.status` → `sold` | Sale celebration, sale price, days on market, seller testimonial prompt |
| `open_house_invite` | Showing/open house created | Date, time, address, RSVP CTA, neighborhood highlights |
| `neighbourhood_guide` | New lead enters CRM (first email in buyer journey) | Local spots, schools, transit score, lifestyle content for the contact's area of interest |
| `home_anniversary` | Annual close anniversary (cron) | Estimated current value, equity gain, home tips, market comparison |

### 2. Claude AI Content Generation
- `generateNewsletterContent(context: NewsletterContext)` in `src/lib/newsletter-ai.ts`
- Context includes: `emailType`, `recipientName`, `contactProfile` (journey phase, interests, behavior score), `newsletter_intelligence` (JSONB from contact)
- Model: Claude Sonnet (via `ANTHROPIC_API_KEY`)
- Returns structured JSON: `{ subject, intro, body, stats?, listings?, recentSales?, ctaText }`
- AI reads full click history and inferred interests before generation — each email is unique per recipient

### 3. React Email Templates
- All 6 templates extend `BaseLayout.tsx` which provides: realtor branding, dark mode, unsubscribe link (`/api/newsletters/unsubscribe?id={contactId}`)
- Templates rendered to HTML via `@react-email/components` `render()` function
- Branding pulled from env vars: `AGENT_NAME`, `AGENT_BROKERAGE`, `AGENT_PHONE`, `AGENT_EMAIL`

### 4. Send Mode: Review vs Auto
- `newsletters.send_mode = "review"` → AI draft lands in approval queue at `/newsletters/queue`
- `newsletters.send_mode = "auto"` → email sent immediately via Resend without human review
- Per-contact and per-workflow configurable via `contact_journeys.send_mode`

### 5. Contact Journeys (Lifecycle-Driven)
- Buyer journey phases: `lead` → `active` (after showing booked) → `under_contract` → `past_client` → `dormant`
- Seller journey phases: `lead` → `active_listing` → `under_contract` → `past_client` → `dormant`
- Phase transitions are event-driven via `workflow-triggers.ts`: `showing_completed`, `offer_accepted`, `deal_closed`
- Each phase maps to different email frequency and content type
- `src/actions/journeys.ts` handles `enrollContact()`, `advancePhase()`, cron-triggered sends

### 6. Visual Automation Builder
- `/automations` renders `WorkflowCanvas` (React Flow) — drag-and-drop sequence editor
- Workflow stored as `flow_json` (React Flow graph) + `workflow_steps` (executable array)
- Step types: `ai_email` (Claude generates content), `auto_email` (template with variable substitution), `wait` (delay in days), `condition` (branch on contact field)
- `src/lib/workflow-engine.ts` executes steps: checks `workflow_enrollments.next_run_at`, sends via Resend, advances to next step
- `src/lib/flow-converter.ts` converts React Flow graph ↔ `workflow_steps` array

### 7. Segments
- `/contacts/segments` renders `SegmentBuilder` component
- `contact_segments.filter_criteria` (JSONB): filter by `type`, `stage_bar`, `lead_status`, `areas`, `price_range`, `behavior_score`
- `src/actions/segments.ts`: `buildSegment()`, `bulkEnroll(segmentId, workflowId)`
- Segment `contact_count` updated on each build

### 8. Click Tracking and Engagement Scoring
- Resend webhook at `/api/webhooks/resend` processes `email.opened`, `email.clicked`, `email.bounced`, `email.unsubscribed`
- Click events stored in `newsletter_events` with `link_url` and `link_type` (`listing_detail`, `book_showing`, `cma_request`, `general`)
- High-intent clicks (`book_showing`, `cma_request`) → realtor alert triggered immediately
- `contacts.behavior_score` updated via `newsletter_events` aggregation
- `contacts.newsletter_intelligence` updated with click history and inferred interests (areas, property types, price range)

### 9. Approval Queue
- `ApprovalQueueClient.tsx` at `/newsletters/queue` shows AI drafts with: subject, preview, recipient segment, confidence score
- Realtor can: Approve (schedules send), Edit (opens template editor), Skip (dismisses draft)
- Bulk actions: approve all, skip all

## End-to-End Scenarios

### Scenario 1: AI Generates New Listing Alert, Realtor Approves and Sends

1. Realtor marks listing at `456 Oak Street` as Active
2. `workflow-triggers` fires `new_listing` event
3. Journey engine identifies all buyer contacts in `active` phase with matching price/area criteria
4. For each matched buyer, `generateNewsletterContent({ emailType: "new_listing_alert", ... })` is called
5. Claude generates personalized subject and listing card content
6. `newsletters` record inserted with `status: "draft"`, `send_mode: "review"`
7. Realtor opens `/newsletters/queue` — draft appears with subject "New Home Alert: 456 Oak, 3BR/2BA – Just Listed"
8. Realtor clicks Preview — sees full rendered React Email HTML
9. Realtor clicks Approve → `newsletters.status` → `"approved"` → Resend sends immediately
10. `newsletter_events` records `email.opened` when buyer opens
11. Buyer clicks "Book a Showing" link → `newsletter_events` records `link_type: "book_showing"` → realtor receives high-intent alert

### Scenario 2: Monthly Market Update Auto-Sent to Farming Segment

1. Cron job fires on the 1st of each month
2. `contact_segments` "Burnaby Buyers" segment evaluated — 47 contacts match criteria
3. For each contact, `generateNewsletterContent({ emailType: "market_update", area: "Burnaby" })` generates unique stats commentary based on contact's `newsletter_intelligence`
4. All 47 `newsletters` records inserted with `send_mode: "auto"` → sent immediately via Resend
5. `/newsletters/analytics` updates: open rate 34%, avg CTR 8%
6. `contacts.behavior_score` updated for all openers and clickers

### Scenario 3: Buyer Journey Phase Advancement

1. Buyer contact "Maria Chen" is in `lead` phase, enrolled in buyer journey
2. Realtor confirms a showing for Maria → `showing_completed` trigger fires
3. `advancePhase(contactId, "active")` called in `src/actions/journeys.ts`
4. `contact_journeys.current_phase` → `"active"`
5. Journey engine switches email cadence from weekly neighbourhood guides to bi-weekly new listing alerts
6. Maria's `newsletter_intelligence` updated with showing data

### Scenario 4: Visual Workflow — New Lead Drip Sequence

1. Realtor opens `/automations`, clicks "New Workflow"
2. Drags nodes onto React Flow canvas: `Trigger: contact_created` → `Wait: 0 days` → `ai_email: neighbourhood_guide` → `Wait: 7 days` → `auto_email: market_update` → `Wait: 14 days` → `ai_email: new_listing_alert`
3. Saves workflow — `flow_json` and `workflow_steps` stored
4. New buyer contact "James Park" added to CRM
5. `bulkEnroll` enrolls James → `workflow_enrollments` record created with `current_step: 0`
6. `workflow-engine.ts` executes step 0 immediately: Claude generates neighbourhood guide for James
7. 7 days later cron advances to step 2: market update sent
8. 14 days later: personalized new listing alert sent

### Scenario 5: Realtor Edits AI Draft Before Sending

1. AI generates Home Anniversary email for past client "Bob Lee"
2. Draft lands in `/newsletters/queue` with `status: "draft"`
3. Realtor reads preview — AI content is good but Realtor wants to add a personal note
4. Realtor clicks Edit → opens `EmailEditorClient` at `/automations/templates`
5. Realtor adds a paragraph about Bob's specific neighbourhood
6. Saves and Approves → `newsletters.status` → `"approved"` → sent via Resend

### Scenario 6: Voice Agent Checks and Approves Newsletter

1. Realtor says: "How many newsletters are waiting for approval?"
2. Voice agent calls `get_newsletters({ status: "draft" })`
3. Agent responds: "You have 3 drafts waiting. The most recent is a New Listing Alert for the Burnaby segment, ready since this morning."
4. Realtor says: "Approve them all"
5. Voice agent calls `approve_newsletter({ newsletter_id: "...", action: "approve" })` for each
6. Agent responds: "Done. All 3 newsletters have been approved and queued for sending."

## Demo Script

**Setup:** Have at least one buyer contact with `newsletter_intelligence` populated, one draft newsletter in the queue.

## Voice Agent Integration

### Available Tools (Realtor Mode)

| Tool | Parameters | Action |
|------|-----------|--------|
| `get_newsletters` | `status` (`draft`/`approved`/`sent`), `limit` | List newsletters by status |
| `approve_newsletter` | `newsletter_id`, `action` (`approve`/`skip`) | Approve or dismiss a draft |
| `get_workflows` | (none) | List all available automation workflows |
| `enroll_in_workflow` | `workflow_id`, `contact_id` | Enroll a contact in a workflow |
| `get_enrollments` | `contact_id` | Check active enrollments for a contact |
| `manage_enrollment` | `enrollment_id`, `action` (`pause`/`resume`/`exit`) | Control an active enrollment |
| `navigate_to` | `page: "newsletters"` \| `"newsletters/queue"` \| `"newsletters/analytics"` \| `"automations"` | Navigate to email marketing pages |

### API Bridge Routes
- `GET /api/voice-agent/newsletters` — list newsletters
- `POST /api/voice-agent/newsletters` — approve/skip action
- `GET /api/voice-agent/workflows` — list workflows
- `POST /api/voice-agent/enrollments` — enroll contact
- `GET /api/voice-agent/enrollments` — check enrollments
- `PATCH /api/voice-agent/enrollments` — pause/resume/exit

### Example Voice Interactions
- "How many email drafts are waiting?" → `get_newsletters({ status: "draft" })`
- "Approve the new listing alert draft" → `get_newsletters` → `approve_newsletter({ action: "approve" })`
- "Enroll Sarah Chen in the buyer drip workflow" → `get_workflows` → `enroll_in_workflow({ workflow_id, contact_id })`
- "Pause James Park's email workflow" → `get_enrollments({ contact_id })` → `manage_enrollment({ action: "pause" })`
- "Take me to the approval queue" → `navigate_to({ page: "newsletters/queue" })`
