Email Content Generation Pipeline

Every email passes through a multi-stage pipeline where AI generates content, text is sanitized, and quality is scored before any email leaves the system.

Trigger Event
Journey cron fires (9 AM UTC), workflow step executes, or realtor clicks "Generate" in the UI. The system determines the email type (listing_alert, market_update, neighbourhood_guide, etc.) based on the contact's journey phase.
Context Assembly
Builds a NewsletterContext object: contact profile (name, type, areas, preferences, engagement score, click history), realtor branding (name, brokerage, phone), active listings (address, price, beds/baths, photos), journey phase, and RAG-retrieved interaction history beyond 30 days.
Claude AI Generation Haiku 4.5
System prompt composed from: base realtor context + contact type instructions (buyer vs seller messaging) + journey phase tone (lead=trust, active=action, dormant=re-engage) + personalization hints from lead scorer (tone, interests, price anchor, hot topics, things to avoid) + contact intelligence (click history, inferred interests) + realtor notes (sanitized, max 400 chars). Returns structured JSON: subject, intro, body, ctaText, highlights, stats, tips, reasoning.
Data Hydration
AI-generated content is enriched with real database data: listing photos, actual market stats (avg price, DOM, inventory), mortgage calculations, testimonials, area highlights, recent sales comparisons. AI provides the narrative; DB provides the numbers.
Text Pipeline (6-Step Sanitization) GATE
1. Personalize: Replace {first_name}, {area}, {budget_min} tokens. Block if unresolved.
2. Voice Rules: Apply realtor's writing style (no exclamation marks, word limits, avoid-words).
3. Content Safety: Detect suspiciously specific claims (>3 prices/addresses).
4. BCFSA/CASL Compliance: Block guarantees, predictions, disparagement, false urgency, unsubstantiated claims, investment advice.
5. Subject Dedup: Check 14-day history for duplicate or similar subjects (first 40 chars).
6. Length Check: Validate word count per email type (listing_alert: 40-100, market_update: 80-200, etc.).
Quality Scoring (7 Dimensions) GATE
Claude Haiku scores 1-10 on: Personalization, Relevance, Data Accuracy, Tone Match, CTA Clarity, Length, Uniqueness. Average = overall score. Decision: score >= 6 = SEND, 4-6 = SEND with warning, < 4 = BLOCK. Any single dimension <= 2 = BLOCK regardless of average.
Block Assembly 13+ blocks
Content mapped to journey-aware block sequences. Example for listing_alert/lead: [header, heroImage, priceBar, personalNote, featureList, photoGallery, priceComparison, cta, agentCard, footer]. High-intent contacts (score >= 70) get a priority booking banner prepended. Themes: standard, luxury, editorial. Dark mode + mobile responsive CSS included.
Validated Send Decision
Final routing: SEND (quality passed + governor allows) -> Resend API -> status="sent". QUEUE (quality passed but needs approval) -> draft for realtor review. DEFER (governor says too soon) -> set next_email_at. BLOCK (quality failed or compliance blocked) -> status="failed" with reasons. REGENERATE (fixable issues) -> flag for retry with adjusted prompts.

Quality Assurance Pipeline

Every email must pass through 3 independent gates before delivery. Any gate can block, defer, or flag an email for human review.

GATE 1 CASL + Compliance

  • Consent required: casl_consent_given=true within 2 years
  • Unsubscribe check: newsletter_unsubscribed=false
  • Physical address in footer (CASL requirement)
  • Unsubscribe link in every email
  • BCFSA blocklist: no guarantees, predictions, disparagement, false urgency, investment advice

GATE 2 Send Governor

  • Phase-aware frequency caps: lead=72h, active=48h, under_contract=24h, past_client=168h, dormant=336h
  • Engagement throttle: declining trend + score < 30 = max 1 per 2 weeks
  • Auto-sunset: 0 opens in 90 days across 5+ emails = pause journey
  • Weekend skip (configurable per realtor)
  • Master sending switch

GATE 3 AI Quality Scorer

  • 7 dimensions scored 1-10 by Claude Haiku
  • Overall >= 6: SEND
  • Overall 4-6: SEND with warning
  • Overall < 4: BLOCK
  • Any dimension <= 2: BLOCK (critical failure)
  • Weekly learning correlates scores with actual engagement

The Engagement Feedback Loop

Every email interaction feeds back into the system, making future emails smarter. This is how cold leads become hot leads -- and how the system learns to stop emailing people who aren't interested.

What Each Interaction Updates

Email Opened

  • Engagement score: +2 points
  • Update last_opened timestamp
  • Increment total_opens
  • Recalculate engagement trend (stable/improving/declining)
  • Emit email_opened event to agent service

Link Clicked

  • Classify link: listing (+15), showing (+30), CMA (+30), mortgage (+20), etc.
  • Update engagement score by category impact
  • Append to click_history[] (keep last 50)
  • Infer interests: school_info -> "family", investment -> "investor"
  • Sync inferred areas + property types to buyer_preferences

High-Intent Click (score >= 25)

  • Create follow-up task for realtor (due in 24h, priority=high)
  • Create HOT LEAD communication record
  • Send agent notification alert
  • Set next_email_type_override (book_showing -> open_house_invite)
  • Advance journey phase: lead -> active, dormant -> lead

Bounce / Complaint

  • Bounce: engagement score -30, pause all journeys
  • Complaint: engagement score -50, pause all journeys, increment complaint_count
  • Unsubscribe: mark newsletter_unsubscribed=true, pause all
  • 5+ emails with 0 opens in 90 days: auto-sunset to dormant

Personalization Hints (AI -> Content Generator)

The lead scorer produces personalization hints that directly influence the AI content generator:

tone

data-driven | warm | formal - Adjusts the writing style of AI-generated content

interests

Array of areas and property types inferred from clicks: ["Kitsilano", "condos", "waterfront"]

price_anchor

The price range the contact is most interested in: "$850K-$1.1M" - Used to select matching listings

hot_topic

What to lead with: "new listings in Kitsilano" or "mortgage rate changes"

avoid

What NOT to include: "generic market updates" or "investment properties"

relationship_stage

cold | warm | hot - Affects formality level and CTA directness

Engagement Score Formula

// Decay: -5 per 30 days after 60 days of inactivity
decay = daysSinceLastOpen < 60 ? 0 : -5 * floor((daysSinceLastOpen - 60) / 30)

// New score = current + decay + event impact
newScore = min(100, currentScore + decay + scoreImpact)

// Trend detection
trend = daysSinceLastOpen > 30 ? "declining"
       : newScore > prevScore + 5 ? "improving"
       : "stable"

Data Sources Feeding the Email Engine

Contact Data

  • Profile: name, email, phone, type (buyer/seller), pref_channel
  • Preferences: areas, property types, budget range, features
  • Intelligence: engagement_score, click_history (last 50), inferred_interests
  • AI Score: buying_readiness, timeline_urgency, budget_fit, intent, personalization_hints
  • Journey: current_phase, emails_sent_in_phase, last_email_at
  • Compliance: casl_consent, newsletter_unsubscribed, complaint_count

Listing Data

  • Property: address, price, beds/baths/sqft, property_type, status
  • Photos: hero_image_url + listing_photos (5 per listing, role-tagged)
  • MLS: mls_number, public_remarks, realtor_remarks
  • Features: feature list, showing instructions, marketing tier
  • History: days_on_market, sold_price, price changes

Market Data

  • Stats: avg price, median DOM, inventory count by area
  • Trends: price change %, absorption rate, months of supply
  • Recent Sales: address, sale price, DOM (for market update emails)
  • Source: Market scraper cron (weekly), BC Assessment API, manual entry

Realtor Data

  • Branding: name, brokerage, phone, email, logo, photo
  • Voice: writing rules extracted from past edits (no exclamation marks, etc.)
  • Config: frequency caps, skip_weekends, sending_enabled, default_send_time
  • Stats: homes sold, volume, years experience (for social proof blocks)

Interaction History

  • Communications: SMS, email, WhatsApp messages (last 20)
  • Newsletter Events: opens, clicks, bounces, complaints (last 30)
  • Showings: confirmed, completed, cancelled (last 30d)
  • RAG: Full semantic search across all history (unlimited window)

External APIs

  • Resend: Email delivery, open/click/bounce tracking
  • Anthropic Claude: Content generation (Haiku), quality scoring, lead scoring
  • BC Assessment: Property valuations (for anniversary emails)
  • Google Calendar: Showing schedule, availability
  • Twilio: SMS/WhatsApp for showing notifications
