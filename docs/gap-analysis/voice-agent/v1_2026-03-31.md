## Summary

The voice agent has a solid core engine — Python aiohttp server with 64 tools, 4 LLM providers with fallback, Supabase integration, and a working React widget using Web Speech API + Edge TTS. However, the two PRDs define 8 phases of work across ~21 weeks, and only the foundational layer (~25-30%) is implemented. Major gaps exist in WebRTC audio (Daily.co), calendar integration, notifications, external assistants, and the multi-tenant OAuth system.

## Foundational Decisions (Context for Future Versions)

These architectural choices affect how gaps are evaluated. If any change, flag as a foundational shift.

| Decision | Current State | Implication |
|----------|--------------|-------------|
| Audio pipeline | Web Speech API (browser) + Edge TTS (server) | NOT Daily.co WebRTC as PRD specifies |
| Database | Supabase (migrated from SQLite) | `supabase_client.py` + `tenant_context.py` are live |
| LLM routing | Anthropic primary, OpenAI/Groq/Ollama fallback | Smart routing: Haiku for simple, Sonnet for complex |
| Session storage | Supabase `voice_sessions` table | SQLite `voice_agent.db` still exists on disk (should be removed) |
| Multi-tenant | Migration 061 created tables, partial code integration | RLS policies defined, tenant_id filtering in queries |
