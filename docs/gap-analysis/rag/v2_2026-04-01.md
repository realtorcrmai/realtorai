## PASS 4: Depth Check — read Pass 3 first

### Areas where Passes 1-3 were surface-level

**Session hijacking vulnerability (NEW):**
`getOrCreateSession(sessionId, userEmail)` retrieves session by ID WITHOUT checking `user_email` matches. User A can access User B's conversation by guessing session IDs.
- Evidence: `conversation.ts` line 75-90 — no ownership check
- Fix: 1 line — `if (existing.user_email !== userEmail) return createSession(...)`

**Embedding staleness (NEW):**
No freshness check anywhere. When a contact or listing is updated, its embeddings remain stale indefinitely.
- `realtime-ingest.ts` has `triggerIngest()` but it's NOT called from any create/update action
- No cron for re-embedding outdated content

**rag_search RPC (NEW):**
The SQL function has NO `realtor_id` parameter. Even after adding the column, the RPC must be updated with `AND e.realtor_id = $realtor_id_param`.
