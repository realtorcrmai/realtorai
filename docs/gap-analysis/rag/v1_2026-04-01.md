## CRITICAL SECURITY ISSUE (P0)

### Multi-tenancy Completely Bypassed

**CMMI: Level 0 (Non-existent) for data isolation**

Every RAG file that accesses the database uses raw `createClient()` with `SUPABASE_SERVICE_ROLE_KEY`, bypassing ALL RLS policies:

| File | Uses Admin Client | Accesses User Data | Tenant Scoped |
|------|------------------|-------------------|--------------|
| `retriever.ts` | ✗ `getAdmin()` | Embeddings, contacts, listings | **NO** |
| `conversation.ts` | ✗ `getAdmin()` | Chat sessions, messages | **NO** |
| `ingestion.ts` | ✗ `getAdmin()` | All source tables, embeddings | **NO** |
| `feedback.ts` | ✗ `getAdmin()` | Feedback, audit logs | **NO** |
| `fallback.ts` | ✗ `getAdmin()` | Full-text search across all data | **NO** |
| `summarizer.ts` | ✗ `getAdmin()` | Contacts, communications, appointments | **NO** |
| `mention-parser.ts` | ✗ `getAdmin()` | Contacts, listings | **NO** |
| `instant-search.ts` | ✗ `getAdmin()` | Contacts, listings, showings | **NO** |
| `feedback-analyzer.ts` | ✗ `getAdmin()` | Feedback patterns | **NO** |
| `retention.ts` | ✗ `getAdmin()` | Deletes sessions/feedback globally | **NO** |

**Schema also missing tenant columns:**
- `rag_embeddings`: NO `realtor_id` column
- `rag_sessions`: Has `user_email` but NO `realtor_id`
- `rag_feedback`: NO `realtor_id`
- `rag_audit_log`: Has `user_email` but NO `realtor_id`

**RLS policies are permissive:**
```sql
CREATE POLICY "Allow all authenticated" ON rag_embeddings FOR ALL USING (true);
```

**Impact:** Any authenticated user can search, read, and modify ANY realtor's RAG data.

**Fix required:**
1. Migration: Add `realtor_id` to all 4 RAG tables + indexes
2. Replace all `getAdmin()` with `getAuthenticatedTenantClient()`
3. Update `rag_search()` RPC to filter by `realtor_id`
4. Update RLS policies: `USING (realtor_id = auth.uid()::uuid)`
