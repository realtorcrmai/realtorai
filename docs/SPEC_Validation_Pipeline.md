# Validation Pipeline & Production Testing — Specification

## Overview

Every email generated by the AI passes through a 7-step validation pipeline before reaching any recipient. A canary system tests emails before real contacts receive them. Production monitoring catches issues that development tests can't.

---

## 7-Step Validation Pipeline

### Step 1: AI Content Generation
Claude generates email content using assembled context (contact + market + realtor data).

### Step 2: Content Validator
| Check | Rule | On Fail |
|---|---|---|
| Name match | "Hi {X}" matches contact.name | Block, log |
| Area match | Email mentions contact's preferred areas | Block, log |
| Budget match | Listed properties within contact's budget range | Block, log |
| Type match | Buyer content for buyers, seller for sellers | Block, log |
| Freshness | All referenced listings still active in DB | Block, remove stale |
| Dedup | Subject not sent to this contact in last 14 days | Block, log |
| Length | Body has 50+ chars of real content | Block, log |
| Merge fields | No raw {{variables}} in output | Block, log |
| Safety | No profanity or inappropriate content | Block, alert |

### Step 3: HTML Renderer
React Email blocks assembled with brand config → rendered to HTML string.

### Step 4: Design Validator
| Check | Rule | On Fail |
|---|---|---|
| HTML valid | Parseable, no broken tags | Fall back to plain text |
| Unsubscribe | Link present in footer | Block (legal requirement) |
| Physical address | Brokerage address in footer | Block (CASL/CAN-SPAM) |
| Images | All img src have alt text | Warning |
| Links | All href are valid URLs (not null/undefined) | Block broken links |
| CTA | Primary button has href | Warning |
| Size | HTML < 100KB (Gmail clips at 102KB) | Trim content |
| Dark mode | Text contrast ratio > 4.5:1 | Warning |

### Step 5: Compliance Gate
| Check | Rule | On Fail |
|---|---|---|
| Not unsubscribed | contact.unsubscribed != true | Block (CASL) |
| CASL consent | Valid and not expired (24 month check) | Block |
| Frequency cap | <= max emails/week for this contact | Defer to next window |
| Min gap | >= 18 hours since last email to this contact | Defer |
| Quiet hours | Not between configured quiet start/end | Defer to morning |
| Not bounced | Email address hasn't hard-bounced | Block |
| Master switch | Realtor's sending is enabled | Block |

### Step 6: Trust Gate
| Trust Level | Action |
|---|---|
| Level 0 (Ghost) | Queue for manual review + manual send |
| Level 1 (Co-pilot) | Queue for 1-tap approval |
| Level 2 (Supervised) | Auto-send, include in daily digest |
| Level 3 (Autonomous) | Auto-send, include in weekly summary |

### Step 7: Send + Verify
- Send via Resend API
- Verify HTTP 200 + message_id returned
- Log to newsletters table (status='sent', resend_message_id)
- On failure: retry once after 30s, then mark 'failed' + alert realtor
- Start tracking webhooks (open, click, bounce)

## Quality Scorer

After Step 1 (generation), before Step 2 (validation):

Second Claude Haiku call scores the email on 7 dimensions (1-10 each):
1. Personalization — uses contact's specific data?
2. Relevance — appropriate for their journey phase?
3. Tone — matches voice rules?
4. Value — recipient would find this useful?
5. CTA clarity — next action is obvious?
6. Length — appropriate for content type?
7. Uniqueness — different from last email sent?

| Average Score | Action |
|---|---|
| >= 7 | Proceed to validation |
| 5-7 | Regenerate with feedback from scorer |
| < 5 | Skip this send, log quality failure |
| Any dimension < 4 | Skip, alert realtor |

Cost: ~$0.0005 per email (Claude Haiku).

## Canary System

### 6 Canary Profiles Per Realtor

| Profile | Purpose | Expected Behavior |
|---|---|---|
| Standard Buyer | Happy path with complete preferences | Receives personalized listing alerts |
| Minimal Contact | Only name + email, no preferences | Receives generic welcome, not personalized |
| Seller | Seller type with property details | Receives seller content, not buyer |
| Dormant | Enrolled 90 days ago, never opens | Re-engagement then auto-sunset |
| Unsubscribed | Marked unsubscribed | Receives NOTHING |
| High Frequency | Maximum engagement | Frequency cap enforced correctly |

### Canary Check Pipeline

For each canary email:
1. **Delivery**: Resend confirms delivery? < 60 seconds? Not spam?
2. **Content**: Correct name? Correct area? No raw variables? Not duplicate?
3. **Rendering**: Playwright screenshots at 375px + 600px + dark mode
4. **Images**: Every img src returns HTTP 200?
5. **Links**: Every href returns HTTP 200? Tracking redirects fire webhooks?
6. **Compliance**: Unsubscribed canary received nothing?

### Failure Response

| Severity | Examples | Action |
|---|---|---|
| CRITICAL | Wrong name, no unsubscribe, sent to unsubscribed, empty body | Pause ALL sending, alert realtor + platform |
| WARNING | Broken image, dark mode issue, slow delivery | Send real emails but flag in daily digest |
| INFO | Minor length variance, slight CSS shift | Log for weekly report |

## Production Monitoring

### Real-Time Alerts
- Bounce rate > 3% in last hour → pause sending
- Unsubscribe rate > 1% in last day → pause + review
- 0% open rate on 10+ emails → deliverability check
- Send failure rate > 5% → Resend API status check
- Content validator rejections spike → AI prompt broken
- Compliance gate blocking > 50% → config issue

### Daily Health Check
- Emails sent vs expected
- Open rate vs 7-day rolling average
- Click rate vs 7-day rolling average
- Bounce rate trend
- Unsubscribe trend
- Failed sends count
- Suppressed count
- Validator rejection count

### Weekly Quality Report
- Sample 10 emails per realtor
- Run quality scorer on each
- Average across 7 dimensions
- Compare to last week
- Flag declining quality
